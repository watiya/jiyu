B. 투표권 영역
B1
내 투표권 요약 바 NEW
STAR 잔액·투표 가중치·참여 건수·진행중 건수 4개 카드 수평 배치
필수
고유
DEX DAO 카드
  • 내 STAR 잔액 (스테이킹 포함)
  • 투표 가중치 (1 STAR = 1표)
  • 참여한 투표 수 / 전체 대비 %
  • 진행중 투표 수 (파랑 강조)
Store DAO 카드
  • 보유 Store Token 수량
  • 정족수 기준 (유통량 2%)
  • 내 비율 (유통량 대비 %)
  • 진행중 Store DAO 수 (보라 강조)
업데이트
  • 지갑 잔액 실시간 반영
  • 스테이킹 변동 즉시 반영
  • 지갑 미연결: "지갑 연결" CTA
투표 가중치: DEX DAO = 보유 STAR 수량. Store DAO = 해당 Store Token 수량. 1토큰 = 1표 원칙. 스테이킹된 STAR도 투표권 포함.

내 투표권 우측 카드 NEW

  • 총 DEX DAO 투표권 (합산)
  • "STAR 추가 스테이킹" CTA
Store DAO 섹션
  • 보유 Store Token별 수량
  • 각 Store DAO 투표권
  • 파란 그라디언트 카드 배경

고유

미보유 상점

C. 안건 카드
C1
안건 카드 (진행중) NEW
상태 pill·SIP 번호·DAO 유형 뱃지·제목·설명·메타·투표 바·내 투표 영역
필수
고유
카드 헤더
  • 상태 pill (진행중 파랑)
  • SIP-012 번호 (monospace)
  • DEX DAO / Store DAO 유형 뱃지
  • 안건 제목 (bold, 14px)
  • 설명 (2줄 clamp)
메타 정보
  • 🗳 참여자 수
  • 📅 마감까지 남은 시간
  • ⚡ 정족수 달성 여부
  • 온체인 제안 해시 (가결 시)
진행중 스타일
  • 테두리: 파랑 (DEX) / 보라 (Store)
  • 배경: 연한 파랑 tint
  • 카드 클릭 → 상세 페이지 이동
C2
안건 카드 (가결·부결·완료)
상태별 스타일 차이. 가결: 초록 / 부결: 빨강 / Timelock: 보라 / 완료: 회색
필수
고유
상태 pill 종류
  • 진행중: 파랑 배경
  • Timelock: 보라 배경
  • 가결·완료: 초록 배경
  • 부결: 빨강 배경
  • 취소: 회색 배경
완료 카드 표시
  • opacity .8 (시각적 구분)
  • 실행 완료 날짜·시간
  • 온체인 트랜잭션 해시
  • 투표 버튼 없음
Timelock 카드
  • 보라 테두리 강조
  • 카운트다운 타이머 표시
  • "자동 실행 예정" 안내
  • 긴급 취소 버튼 (멀티시그)
C3
투표 결과 바 NEW
찬성(초록)·반대(빨강)·기권(회색) 3색 진행 바 + 퍼센트 + STAR 수량
필수
고유
바 구성
  • 가로 바 (찬성 + 반대 + 기권 = 100%)
  • 찬성: 초록 (좌측)
  • 반대: 빨강 (우측)
  • 기권: 회색 (사이)
  • 정족수 기준선 마커 (세로선)
수치 표시
  • 찬성 % + STAR 수량 (좌측)
  • 기권 % (중앙)
  • 반대 % + STAR 수량 (우측)
  • 총 참여 투표수
가결 판정
  • 정족수 AND 과반수 동시 충족 시 가결
  • 정족수 미달 시 점선 바 + 경고
  • 투표 종료 후 최종 결과 확정
가결 조건: DEX DAO — 전체 STAR의 5% 이상 참여(정족수) AND 찬성 과반수. Store DAO — 유통량 2% 이상 참여 AND 찬성 과반수. 5일 투표 후 확정.
C4
Timelock 대기 배너 NEW
가결 안건 48시간 Timelock 대기 알림 + 실시간 카운트다운 + 긴급 취소 안내
필수
고유
배너 구성
  • ⏳ 아이콘
  • 안건명 + "Timelock 대기 중"
  • 실시간 카운트다운 (HH:MM:SS)
  • "자동 실행 예정" 설명
안전 장치
  • 문제 발견 시 48시간 내 취소 안내
  • 취소: 멀티시그 서명 필요
  • 보라 배경으로 주의 환기
Timelock 목적: 가결 후 즉시 실행이 아닌 48시간(DEX) / 24시간(Store) 대기. 이 기간 내 버그·문제 발견 시 멀티시그로 취소 가능. 커뮤니티 안전 장치.
C5
안건 상세 페이지 NEW
개별 안건 전체 내용·투표 현황·토론·온체인 실행 데이터 상세 페이지
필수
dYdX 참고
좌측: 안건 본문
  • 전체 제목 + 상태
  • 제안자 주소 + 제안일
  • 전체 설명 (마크다운 렌더링)
  • 실행 시 파라미터 코드 블록
  • 타임라인 (제안→투표→Timelock→실행)
중앙: 투표 현황
  • 찬성/반대/기권 대형 투표 바
  • 정족수 달성 여부 + 진행 바
  • 투표자 목록 (주소 단축 + 수량)
  • 내 투표 버튼 (찬성/반대/기권)
우측: 요약 정보
  • 안건 ID + 상태
  • 투표 시작·마감 날짜
  • 제안자 주소
  • 온체인 해시 링크
  • 관련 안건 링크

D1

필수
고유
버튼 구성

  • 👎 반대 (빨강 테두리)
  • — 기권 (회색 테두리)
  • hover: 배경색 채우기
투표 전 상태

  • 투표 바 실시간 업데이트
온체인 서명: 투표 클릭 → Passkey 지문 인증 → UserOperation 생성 → Bundler 제출 → 온체인 기록. 가스비 = Paymaster 대납 (사업주와 동일). 사용자 투표 비용 = 0.

고유

F. 거버넌스 파라미터 영역
F1
거버넌스 파라미터 카드 NEW
투표 기간·정족수·가결 조건·Timelock·제안 요건. DEX/Store 구분 표시
필수
고유
DEX DAO 파라미터
  • 투표 기간: 7일
  • 정족수: 전체 STAR의 5%
  • 가결 조건: 찬성 과반수(50%+1)
  • Timelock: 48시간
  • 제안 요건: 10,000 STAR 보유
