설계 원칙
나라 → 도시 아코디언 트리
나라 클릭 시 도시 목록 펼쳐짐. 나라 선택만으로도 해당 국가 전체 필터링. 도시 선택 시 해당 도시만 필터링. 현재 선택 상태 강조 표시.
상점 수 카운트 실시간 표시
각 나라·도시 옆에 상장 상점 수 표시. 필터 적용 시 카운트가 해당 조건에 맞는 수로 실시간 변경. 0개인 나라는 비활성 처리.
URL 파라미터 연동
/markets?country=KR&city=seoul 형태. 링크 공유 시 필터 상태 그대로 전달. 뒤로가기 시 복귀 보장.
ISO 3166-1 alpha-2 국가 코드
KR, JP, US, SG, TW, HK, TH 등 표준 코드 사용. 국기 이모지 Unicode Regional Indicator 자동 렌더링. 한국어+영어 나라명 지원.

검색 대상 — 글로벌 축 포함

상점명
필수 한글·영문 모두. 부분 일치 지원. 예: "브루밍", "Bruming"
토큰 심볼
필수 영문 3~5자. 대소문자 무시. 예: "CAFE", "cafe"
나라명
필수 한국어+영어 모두. "korea", "일본", "japan", "미국", "usa", "KR" 등 ISO 코드도 지원
도시명
필수 "서울", "seoul", "도쿄", "tokyo", "san francisco", "싱가포르" 등 한·영 모두
지역구·동네
높음 도시 내 하위 지역. "성수동", "시부야", "mission district", "홍대" 등
업종 카테고리
높음 "카페", "음식점", "피트니스" 등. 업종 전체 결과 페이지로 연결
복합 자연어 쿼리
Starzip 고유 "도쿄 카페", "서울 피트니스", "tokyo cafe", "singapore hair" 등 나라+업종

핵심 인사이트 — Hyperliquid와 dYdX의 검색은 "심볼을 이미 아는 사람"을 위한 도구입니다. Starzip의 검색은 반대로 "어떤 상점이 있는지 탐색하는 사람"을 위한 도구입니다. "홍대 카페", "도쿄 피트니스" 같은 자연어 쿼리를 지원하는 것이 크립토 DEX 중 유일한 차별화 포인트입니다.
글로벌 위치 구조는 상점 수에 따라 단계적으로 도입하는 것이 현실적입니다. 초기에 모든 나라 트리를 만들어도 대부분 비어있으면 오히려 신뢰를 낮춥니다.

Phase 1 · 런치 (현재)

  • 한국만 서비스
  • 사이드바: 서울·부산·제주 도시만
  • 검색: 상점명·심볼·지역구
  • URL: ?city=seoul 수준
  • 카드: "성수동 · 서울" 표기
  • 위치 칩 없음 (단순 검색창)

Phase 2 · 아시아 확장

  • KR + JP + SG + TW
  • 사이드바: 나라 아코디언 도입
  • 검색: 나라명·도시명 추가
  • 국기 이모지 + ISO 코드
  • 카드: "성수동 · 서울 · 🇰🇷" 표기
  • 위치 칩 검색창에 추가
  • 카테고리 현지화 전략 수립

Phase 3 · 글로벌

  • 전체 글로벌 오픈
  • GPS 자동 위치 감지
  • 복합 자연어 쿼리 지원
  • "tokyo cafe" 검색 지원
  • 나라별 랭킹 리더보드
  • 검색 결과 지도 뷰 (선택)
  • 현지 업종 태그 (이자카야 등)
Phase 2 진입 전 필수 결정사항 — 나라가 늘어날수록 카테고리 태그 현지화가 필요합니다. 일본의 "居酒屋(이자카야)", 싱가포르의 "Hawker Center" 같이 현지 업종 개념이 글로벌 카테고리 체계에 맞지 않을 수 있습니다. Phase 2 진입 전에 카테고리 현지화 전략도 함께 정의해야 합니다.
UX 플로우 — 검색 단계별
1

검색 진입

검색창 클릭 또는 Cmd+K 단축키. 마켓 어느 화면에서도 접근 가능. 진입 즉시 드롭다운 오픈 — 최근 검색어 + 업종/지역 빠른 탐색 표시.

트리거: 클릭 · Cmd+K · 모바일 탭
2

입력 전 — 빠른 탐색 제공

최근 검색어 + 업종 카테고리 칩 + 나라/도시 목록. 검색어 없이도 카테고리·나라로 바로 진입 가능.

표시 조건: 검색창 포커스 & 입력값 없음
3

입력 중 — 실시간 자동완성

1자 이상 입력 시 디바운스 150ms 후 API 호출. 현재 위치 상점 / 나라 전체 / 글로벌 같은 업종 3섹션으로 결과 표시. 검색어 하이라이트.

트리거: keyup (디바운스 150ms)
4a

결과 선택 — 상점 직접 클릭

드롭다운 내 상점 항목 클릭 → /trade/CAFE 거래 화면으로 직접 이동. 가장 빠른 경로.

결과: /trade/[TOKEN] 페이지 이동
4b

결과 선택 — 나라/도시 클릭

나라 또는 도시 항목 클릭 → /markets?country=JP 또는 ?city=tokyo로 이동. 사이드바 트리 자동 동기화.

결과: /markets?country= 또는 ?city= 이동
4c

결과 선택 — Enter / 전체 결과 보기

Enter 또는 전체 결과 보기 → /markets?q=[검색어]. URL 반영으로 공유 가능, 뒤로가기 복귀 가능.

결과: /markets?q=[검색어] 이동
5

결과 없음 처리

매칭 결과 0개 → 유사 업종 칩 제안 + "상점을 직접 제안하기" CTA. 글로벌의 경우 인접 나라 추천도 함께 표시.
